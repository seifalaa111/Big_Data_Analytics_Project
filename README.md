# Real-Time Credit Card Fraud Detection Pipeline

A production-style machine learning pipeline that detects fraudulent credit card transactions in real time using **Apache Kafka**, **XGBoost**, **Apache Spark**, and **Streamlit**.

## System Architecture

```
+----------------+       +----------------+       +-------------------+
|                |       |                |       |                   |
|  creditcard    | ----> |  Kafka Topic   | ----> |  consumer.py      |
|  .csv          |       | "transactions" |       |  (XGBoost Model)  |
|                |       |                |       |                   |
+----------------+       +-------+--------+       +--------+----------+
   producer.py                   |                          |
   streams rows                  |                     results.csv
   to Kafka                      |                     fraud_alerts.log
                                 |                          |
                                 v                          v
                     +---------------------+     +---------------------+
                     | spark_streaming.py   |     |    app.py           |
                     | (Spark Structured    |     |    (Streamlit       |
                     |  Streaming Consumer) |     |     Dashboard)      |
                     +---------------------+     +---------------------+
```

## Components

| Component | File | Description |
|-----------|------|-------------|
| **Producer** | `producer.py` | Reads CSV and streams transactions to Kafka |
| **Consumer** | `consumer.py` | Kafka consumer that classifies transactions with XGBoost |
| **Dashboard** | `app.py` | Real-time Streamlit dashboard with live metrics |
| **Spark Batch** | `spark_analysis.py` | PySpark batch analytics on the full dataset |
| **Spark Streaming** | `spark_streaming.py` | Spark Structured Streaming consumer from Kafka |
| **Model Training** | `Fraud_Detection_Training.ipynb` | Colab notebook: EDA, SMOTE, XGBoost vs RF |
| **Config** | `config.py` | Centralized configuration for all components |

## Tech Stack

- **Message Broker:** Apache Kafka (Docker)
- **ML:** XGBoost, Scikit-learn, SMOTE (imbalanced-learn)
- **Big Data:** Apache Spark (Batch + Structured Streaming)
- **Dashboard:** Streamlit + Plotly
- **Data:** Pandas, NumPy

## Dataset

- **Source:** [Kaggle Credit Card Fraud Detection](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud)
- **Size:** 284,807 transactions, 31 features
- **Imbalance:** Only 0.172% are fraud (492 out of 284,807)
- **Features:** Time, V1-V28 (PCA), Amount, Class

## Setup & Run

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Start Kafka
```bash
docker-compose up -d
```

### 3. Train Model (Google Colab)
- Upload `Fraud_Detection_Training.ipynb` and `creditcard.csv` to Colab
- Run all cells
- Download `model.pkl` and place it in this folder

### 4. Run the Pipeline (3 terminals)
```bash
# Terminal 1 - Start consumer
python consumer.py

# Terminal 2 - Start producer
python producer.py

# Terminal 3 - Start dashboard
streamlit run app.py
```

### 5. Spark Analysis (bonus)
```bash
# Batch analysis
python spark_analysis.py

# Streaming consumer (alternative to consumer.py)
python spark_streaming.py
```

## Key Design Decisions

- **SMOTE over random oversampling:** Generates synthetic minority samples rather than duplicating existing ones, reducing overfitting risk
- **AUPRC over accuracy:** With 99.83% normal transactions, accuracy is misleading. AUPRC properly evaluates minority class detection
- **Sampling strategy 0.1:** Generates fraud samples to 10% of normal count, avoiding over-generalization from a 1:1 ratio
- **Dual Spark integration:** Both batch analytics and real-time structured streaming demonstrate scalability

## Output Files

| File | Generated By | Purpose |
|------|-------------|---------|
| `model.pkl` | Colab notebook | Serialized XGBoost model |
| `results.csv` | `consumer.py` | All classified transactions |
| `fraud_alerts.log` | `consumer.py` | Fraud-only alert log |
